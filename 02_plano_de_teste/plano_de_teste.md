# Plano de Teste – Sistema SRL

## 1. Escopo
Este plano de teste contempla a verificação das principais funcionalidades do **Sistema SRL**, garantindo que o sistema funcione conforme o esperado, de acordo com os requisitos definidos.

Serão testados:
- Funcionalidades principais do sistema
- Regras de negócio
- Restrições e validações
- Comportamento do sistema diante de entradas válidas e inválidas
  
---

## 2. Objetivo
O objetivo deste plano de teste é:
- Identificar erros e falhas antes da entrega do sistema
- Garantir que o Sistema SRL atenda aos requisitos funcionais
- Verificar se as regras e restrições estão sendo aplicadas corretamente
- Aumentar a confiabilidade e a qualidade do sistema

---

## 3. Estratégia de Teste
A estratégia adotada será baseada principalmente em **testes funcionais**, realizados de forma manual.

Serão utilizados:
- **Testes de caixa preta**: foco no comportamento do sistema, sem análise interna do código
- **Testes de validação**: verificação de campos obrigatórios, formatos e limites de dados
- **Testes de fluxo**: validação do caminho correto das funcionalidades

Os testes serão realizados a partir de **casos de teste previamente definidos**, baseados nos requisitos do sistema.

---

## 4. Ambiente
Os testes serão executados no seguinte ambiente:

- Sistema Operacional: Windows
- Ambiente de teste: Postman

---

## 5. Critérios de Entrada
Os testes só poderão ser iniciados quando:
- O ambiente estiver corretamente configurado
- O código do sistema estiver compilando sem erros
- As funcionalidades a serem testadas estiverem implementadas
- Os requisitos do sistema estiverem definidos e documentados

---

## 6. Critérios de Saída
Os testes serão considerados finalizados quando:
- Todos os casos de teste planejados forem executados
- Não existirem erros críticos ou bloqueadores
- Os erros encontrados forem corrigidos ou devidamente documentados
- O sistema estiver funcionando conforme os requisitos definidos
