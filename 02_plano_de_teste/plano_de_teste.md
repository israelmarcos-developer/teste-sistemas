# Plano de Teste – Sistema SRL

## 1. Escopo

Testes funcionais da API REST do SRL cobrindo os quatro módulos descritos nos requisitos.

## 2. Objetivo

Garantir que a API atenda a todos os Requisitos Funcionais e valide corretamente as Regras de Negócio documentadas. 

## 3. Estratégia de Teste

Execução de Testes Funcionais de API:

- Envio de requisições HTTP para os endpoints do sistema simulando o comportamento de clientes reais.
- Validação dos status HTTP retornados.
- Serão realizados testes com entradas válidas e testes de exceção baseados na matriz de rastreabilidade.

## 4. Ambiente

Servidor de teste com a API em execução e ferramentas de consumo para envio das requisições e validação das respostas.

## 5. Critérios de Entrada

- Endpoist constrúidos e disponíveis no ambiente de teste.
- Casos de teste revisados e aprovados.

## 6. Critérios de Saída

Execução completa dos casos de teste planejados e documentação de divergências ou falhas encontradas durante os testes.

## 7. Responsáveis

Desenvolvedor e analísta de QA.
